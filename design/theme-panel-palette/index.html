Color Palette — 6 Integrations

Six ways to bring the deck's color palette (the semantic ColorToken set — Background, Surface, Accent, Text, On-surface, On-accent, Muted) into the existing theme panel, while keeping its compact two-view design, status model, and compact color picker intact. Each keeps the recognizable swatch-palette look; they differ only in placement and emphasis.

Unchanged in all six

Same toolbar Theme button → compact popover → preset view + customize view; same Preset / Custom · Unsaved / Saved status model; same compact color picker; same per-role typography. Only the palette block changes (a single pluggable region), so the panel stays consistent.

Built on the final theme-panel/ design. Shared logic lives in panel-core.js (palette region is pluggable); palette data in theme-model.js (PALETTE_TOKENS, PALETTE_SCHEMES). Mockups are illustrative HTML/JS only.
